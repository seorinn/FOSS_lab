# 중간고사 해설
```
컴퓨터 시스템을 구성하는 요소 4가지: 사용자, 하드웨어, 소프트웨어, 데이터
하드웨어 시스템 구성요소 5가지: CPU, Input Devices, Memory, Output Devices, Storage Devices
터미널에서 기본적으로 사용할 수 있도록 정의된 입출력 3가지: stdin, stdout, stderr
응용프로그램은 직접 하드웨어를 제어하지 않는다.
시스템 소프트웨어에는 운영체제가 포함된다.
터미널은 한번에 캐릭터(문자) 단위로 입출력을 수행한다.
프로세스가 한번 실행 되도 중단 가능하다.
리눅스에서는 장치에 대한 권한을 file에 대한 권한과 동일한 방법(chmod)으로 변경할 수 있다.
dir1로 가는 soft link sdir1만들기: ln -s dir1 sdir1(-s가 빠지면 하드링크)
디렉토리 did의 모든 파일과 디렉토리 삭제: rm -r did
{ ls | wc -l } > wc_result ; cat wc_result 했을 때 ls보다 wc_result가 먼저 생겨서 +1의 값이 출력
wc_result가 이미 있으면 +1되지 않음
| : stdout만 wc -l에 들어감. |& : stdout과 stderr 다 wc -l에 들어감
cat < aaa > ccc : aaa의 내용을 ccc에 복사
리디렉션할 때는 파일 생성이 먼저 이루어짐
1에서 9999까지 수 중에 3의 배수 더함: sum=0. for I in {3..9999..3}. do. sum=$((sum+$i)). done.
```

# 기말고사 정리
```
내가 오픈소스를 쓰는 사람. 리눅스의 오픈소스를 가져다 무엇인가 만듦. 리눅스는 GPL License 그 오픈소스를 사용하는 나는 기본권 같이 갖는 게 사용, 수정, 복제, 배포권. 이 네 가지가 빠져 있으면 오픈소스가 아님. 
Gpl license는 상업적 이용이 불가능. 무언가 만들었을 때 나도 gpl로 소스를 공개해야 함.
Mit license는 상업적 이용이 가능함. 소스 공개의 의무가 없음.
어떤 개발된 공개 소프트웨어를 사용(소스코드 공개). 고쳐서 씀. -> 수정에 대한 고지 의무가 있음
오픈소스로 공개했던 것에 내한 특허소송 불가. Gpl이나 mit 라이센스를 선언했으면 문제없는데 다른 라이선스를 사용한 것 소스코드는 잘못 써서는 안됨. 
이런 가능성이 있는 소스코드를 사용할 경우 몰랐더라도 소송 당할 수 있음
오픈소스에서 뭘 수정하고 특허를 내면 처음 보장된 4대 권리는 무효화됨
사람들이 소프트웨어를 오픈소스로 공개하는 이유: maintenance 비용이 많이 듦. 입소문으로 마케팅.
Engineering service 등 서비스 유료로 해서 돈 벌 수 있음
남의 오픈 소스 코드에 대한 교육을 유료로 해도 됨
내 것일 때만 가능한 것: branded merchandise(로고나 심볼)
소스코드를 가져다 쓰고 특허가 있는 부분을 오픈소스하고 싶지 않을 때는 버전을 다르게 하고 오픈소스를 사용하지 않고 스크래치부터 다시 짜서 유료버전이라 선언할 수 있음
오픈소스로 공개했던 것에 유료 버전을 만드는 것 가능
특정 하드웨어에서만 사용할 수 있는 라이선스는 GPL 3.0에서 금지됨
```
