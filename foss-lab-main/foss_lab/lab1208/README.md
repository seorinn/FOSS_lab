```
기말고사
Hype Cycle을 내고 상식적인 문제
가장 인기: peak of inflated expectation
표준화는 어느 시점에서 일어나는 게 가장 좋을까: slope of enlightenment
-> 이 때 표준화, 시장이 파편화 그리고 오픈소스 프로젝트가 쏟아져 나와 쓸 만한 오픈소스 출시
거의 모든 것을 할 수 있는 수준: plateau of productivity
```
```
소프트웨어 비즈니스
소프트웨어는 만들고 돈을 버는 방법: 첫 번째는 패키지를 판다. 두 번째는 system integration(시스템 통합). 세 번째는 SaaS(시스템 관리)
서비스 관점에서 보면 turn-key based(제일 높은 수준), outsourcing(stage – requirement, design, coding, testing, audit, maintenance), manpower dispatch(제일 낮은 수준: 인력 파견), partnership

소프트웨어 개발 비용
구현 비용보다 테스트비용이 두 배 듦. Maintenance 비용이 가장 큼(예측 힘듦)
Unit test: 여러 명이 프로그램을 짤 때 내가 맡은 함수에 대한 테스트
System test: 전체에서 돌리는 테스트
시스템을 정의하는데 가장 중요한 개념: 어떤 기능의 목적이 있어야 하고 boundary 개념이 있어야 함
소프트웨어 비용을 낼 때 노력 기반의 비용, 시간 기반의 비용이 있음
Cost Factor: 기술적/비기술적
기술적 – module independence, programming language, programming style, program validation and testing, documentation, configuration management
비기술적 – application domain, staff stability, program lifetime, dependence on external environment, hardware stability
```
